# AI 战略分析语音决策设计

## 目标

让 AI 决定每次建议采用简短播报还是完整播报，并在完整播报中加入战略分析。所有语音都在本地通过 Windows SAPI 播放，完整播报目标时长不超过 25 秒，默认语速为 `Rate=4`。

## 输出协议

AI 必须返回严格 JSON：

```json
{
  "analysis": "战略分析",
  "command": "战术指令",
  "item": "出装建议",
  "speech_level": "brief"
}
```

`speech_level` 只有两个合法值：

- `brief`：播放战术指令和出装建议。
- `full`：播放战略分析、战术指令和出装建议。

AI 仅在以下情况选择 `full`：

- 局势或战略方向发生明显变化；
- 出现首次值得说明的关键克制关系；
- 游戏进入新的主要阶段；
- 重大团战、击杀或资源争夺改变了后续策略。

常规重复建议、局势变化很小或只需立即执行短指令时选择 `brief`。

缺少 `speech_level`、值类型错误或值不在允许范围内时，本地统一降级为 `brief`，避免意外播放冗长内容。

## 组件与职责

### 提示词构建器

`PromptBuilder` 更新输出说明和示例，要求模型返回 `analysis`、`command`、`item`、`speech_level` 四个字段，并明确 `brief/full` 的决策标准。

### API 客户端

`AdvisorClient` 解析并校验 `speech_level`，返回包含四个值的结果。旧模型输出缺少该字段时使用 `brief`。非 JSON 回复继续作为战术指令处理，并使用 `brief`。

### AI 事件

`AdvisorEvent` 增加播报等级字段，使模型决策能够从后台 AI 线程传递到 `GSIHandler`。完整分析继续输出到控制台和日志，不因语音裁剪而改变。

### 语音文本组装器

在 TTS 模块中增加可独立测试的文本组装逻辑，输入分析、指令、出装、播报等级与时长配置，输出最终播报文本。

组装顺序为：

1. 战略分析；
2. 战术指令；
3. 出装建议。

`brief` 跳过第一项。空字段直接跳过，不生成空标签。

## 时长与裁剪策略

完整播报的目标上限为 25 秒。默认 SAPI 语速为 `Rate=4`，本地按可配置的“每秒汉字数”估算语音时长。

文本预算遵循以下优先级：

1. 战术指令必须完整保留；
2. 出装建议必须完整保留；
3. 剩余预算用于战略分析。

若完整内容超出预算，只缩短战略分析。裁剪优先寻找预算内最后一个句号、分号、问号、感叹号或逗号等自然边界；没有合适边界时才按字符上限截取。裁剪后的分析不得生成空标签或孤立标点。

若战术指令和出装建议本身已超过预算，仍完整保留它们。25 秒是完整战略分析的目标预算，不以损坏行动指令为代价强行满足。

## SAPI 播放

PowerShell 脚本接收可配置语速，默认 `Rate=4`，并将值限制在 SAPI 支持的 `-10` 到 `10` 范围。

Python 侧的子进程超时根据估算语音时长动态计算，并包含启动与结束缓冲。超时必须高于目标播放时长，避免当前固定 30 秒超时在正常结尾前强制终止。播放超时或 SAPI 异常只记录错误并继续处理下一条语音，不能终止语音工作线程。

## 配置

在 `config.yaml` 中增加语音配置，建议默认值：

```yaml
tts:
  rate: 4
  full_max_seconds: 25
  estimated_chars_per_second: 7
  timeout_buffer_seconds: 8
```

配置读取时进行边界校验：

- `rate` 限制到 `-10..10`；
- `full_max_seconds` 必须为正数；
- `estimated_chars_per_second` 必须为正数；
- `timeout_buffer_seconds` 不得为负数。

非法配置回退到默认值。

## 数据流

```text
GSI 帧
  → AI 提示词
  → AI JSON
  → AdvisorClient 校验 speech_level
  → AdvisorEvent
  → GSIHandler 选择 brief/full
  → 本地语音文本组装与分析裁剪
  → SpeechQueue
  → PowerShell / Windows SAPI
```

## 会话与队列行为

新局开始时清除上一局尚未播放的 AI 战略语音，避免旧局分析在新局继续播报。当前正在播放的 SAPI 进程不做强制中断，防止普通会话重置引入额外进程管理复杂度；清理范围仅为尚未开始的待播队列。

现有视野、资源与昼夜提醒继续使用同一串行队列。本次不引入语音优先级抢占机制。

## 容错

- 缺少或非法 `speech_level`：降级为 `brief`。
- 缺少 `item`：只播放可用的指令，`full` 时仍可播放分析。
- 缺少 `command`：沿用现有兼容规则，把分析转为指令并按 `brief` 播放。
- 非 JSON 模型回复：全文作为指令并按 `brief` 播放。
- 空回复：不创建建议事件，不播放语音。
- TTS 超时或异常：记录错误，语音线程继续运行。

## 测试

虽然项目原先没有统一测试命令说明，仓库已有 `unittest` 测试。本功能继续使用标准库 `unittest`，并按测试驱动方式实现。

测试覆盖：

- 客户端正确解析 `brief` 和 `full`；
- 缺失、非法或错误类型的播报等级降级为 `brief`；
- 非 JSON 回复按 `brief` 兼容；
- `AdvisorEvent` 正确传递播报等级；
- `brief` 只包含指令和出装；
- `full` 包含分析、指令和出装；
- 空字段不会产生空标签；
- 25 秒预算只裁剪分析；
- 裁剪优先发生在中文自然标点；
- 指令和出装即使超过预算也不被截断；
- 默认 `Rate=4` 正确传给 PowerShell；
- 动态超时包含播放时长与缓冲；
- 新局清除上一局尚未播放的 AI 语音；
- 现有 AI 后台线程、触发器和日志测试继续通过。

## 不在本次范围

- 不增加语音优先级或抢占播放；
- 不让本地规则重新判断 AI 的 `brief/full` 决策；
- 不改变 AI 查询频率和触发条件；
- 不裁剪日志或控制台中的完整战略分析；
- 不更换 Windows SAPI 或引入新的语音服务。
